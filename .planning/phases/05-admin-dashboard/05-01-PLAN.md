# Plan 05-01: Admin Base & Analytics

## Goal
Establish the protected `/admin` route structure, create the reusable Sidebar navigation based on `admin.html`, and implement the BI Analytics dashboard.

## Tasks

1. **Admin Layout & Route Protection (`src/app/admin/layout.tsx`)**
   - `<read_first>`: `admin.html`, `src/utils/supabase/client.ts`
   - `<action>`: Create a Next.js layout for the `/admin` route group. Implement a client-side (or server-side) check using Supabase Auth to ensure the user is logged in. If not logged in, redirect to `/auth/login`. Wrap the children with a flex layout containing the new Sidebar component.
   - `<acceptance_criteria>`: `src/app/admin/layout.tsx contains redirect('/auth/login') or router.replace`.

2. **Sidebar Component (`src/components/admin/Sidebar.tsx`)**
   - `<read_first>`: `admin.html`
   - `<action>`: Extract the `<aside id="sidebar">` from `admin.html`. Convert it to a React component. Use Next.js `<Link>` for navigation instead of JavaScript tab switching. Map links to `/admin`, `/admin/catalogo`, `/admin/promocoes`, etc. Include the logout button logic.
   - `<acceptance_criteria>`: Component renders a sidebar with 5 navigation links and a logout button.

3. **Analytics Dashboard (`src/app/admin/page.tsx`)**
   - `<read_first>`: `admin.html`, `src/lib/database/orders.ts`

Calculate and display: Faturamento Total, Garrafas Vendidas, Total de Pedidos. (For the MVP, date filtering can be skipped or implemented basic logic).
   - `<acceptance_criteria>`: Page fetches orders and displays calculated metrics.
