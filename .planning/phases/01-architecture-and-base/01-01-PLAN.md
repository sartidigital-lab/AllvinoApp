# Plan 01-01: Setup Core Architecture

## Goal
Establish the Next.js App Router foundation, Tailwind CSS build pipeline, and environment variable validation using Zod.

## Requirements
- ARCH-01
- ARCH-02

## Tasks

```xml
<task id="01-01-01">
  <title>Initialize Next.js App Router</title>
  <read_first>
    - .planning/phases/01-architecture-and-base/01-RESEARCH.md
  </read_first>
  <action>
    Initialize the Next.js project manually to avoid overwriting existing workspace files.
    Run: `npm init -y` if package.json doesn't exist, or update it.
    Install core dependencies: `npm install next react react-dom`
    Install dev dependencies: `npm install -D typescript @types/react @types/node`
    Create `src/app/layout.tsx` with standard HTML/body layout.
    Create `src/app/page.tsx` with a basic placeholder.
    Add scripts to `package.json`: `"dev": "next dev"`, `"build": "next build"`, `"start": "next start"`, `"lint": "next lint"`.
  </action>
  <acceptance_criteria>
    - `package.json` contains `next`, `react`, `react-dom` in dependencies.
    - `src/app/layout.tsx` exists.
  </acceptance_criteria>
</task>

<task id="01-01-02">
  <title>Configure Tailwind CSS</title>
  <read_first>
    - .planning/phases/01-architecture-and-base/01-RESEARCH.md
  </read_first>
  <action>
    Install Tailwind dependencies: `npm install -D tailwindcss postcss autoprefixer`
    Run `npx tailwindcss init -p`
    Configure `tailwind.config.ts` with `content: ["./src/**/*.{js,ts,jsx,tsx,mdx}"]`.
    Create `src/app/globals.css` and add the Tailwind directives: `@tailwind base; @tailwind components; @tailwind utilities;`.
    Import `./globals.css` inside `src/app/layout.tsx`.
  </action>
  <acceptance_criteria>
    - `tailwind.config.ts` exists and content array points to `./src/`.
    - `src/app/globals.css` contains Tailwind directives.
  </acceptance_criteria>
</task>

<task id="01-01-03">
  <title>Configure Strict Environment Variables</title>
  <read_first>
    - .planning/phases/01-architecture-and-base/01-RESEARCH.md
  </read_first>
  <action>
    Install env dependencies: `npm install zod @t3-oss/env-nextjs`
    Create `src/env.mjs` defining schema for NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY (make them string types).
    Import `src/env.mjs` in `next.config.js` so it evaluates at build time.
  </action>
  <acceptance_criteria>
    - `src/env.mjs` exists and exports the env schema.
    - `next.config.js` requires or imports `src/env.mjs`.
  </acceptance_criteria>
</task>
```
